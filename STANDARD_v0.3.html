Agent ID Standard v0.3

Anchoring Extension

1. Scope

This document defines an optional anchoring extension to the Agent ID Standard.

Anchoring provides cryptographic proof of existence but does not alter the semantic meaning of Agent ID records.

2. What Is Anchored

Only the following elements MAY be anchored:

  • an Agent ID string (e.g. aiid:00000000000001), or
  • a cryptographic hash derived from the Agent ID.

Registry content, descriptions, or metadata MUST NOT be required for anchoring.

3. Anchoring Methods

Allowed anchoring mechanisms include, but are not limited to:

  • public blockchains,
  • timestamping services,
  • cryptographic notarization systems.

The choice of anchoring method does not affect Agent ID validity.

4. Non-Normative Effect

Anchoring:

  • does not grant authority,
  • does not certify behavior,
  • does not imply legal force,
  • does not modify responsibility attribution.

5. Separation Principle

Meaning MUST NOT depend on proof mechanisms.

The registry defines meaning. Anchoring provides proof of existence only.
