Agent ID Position Statement

Agent ID as a Declarative Ontological Layer for AI Agents

Agent ID is not an authentication protocol.
It is not an authorization framework.
It is not an access-control mechanism.

Agent ID defines a declarative ontological layer for artificial agents.

1. The Problem

As AI systems evolve toward autonomy, existing identity infrastructures focus primarily on:

  • authentication (who can log in),
  • authorization (what permissions are granted),
  • access control (what resources may be used).

These mechanisms are necessary — but insufficient.

They answer:

“Can this agent access a system?”

They do not answer:

“What is this agent as a persistent socio-technical actor?”

2. The Missing Layer

Before authentication,
before authorization,
before token issuance —

there must exist a stable referent.

An agent must be:

  • identifiable across contexts,
  • attributable to a responsible principal,
  • distinguishable from implementation details,
  • persistent across platforms.

This is an ontological requirement, not a security feature.

Agent ID addresses this layer.

3. Core Distinction

Layer Function
Identity Ontology (Agent ID) Defines existence
Authentication (OIDC, OAuth, etc.) Verifies session
Authorization Grants permissions
Governance Regulates behavior

Agent ID operates at the level of existence, not session or permission.

3.1 Conceptual Architecture

Layer Role Technology Examples
Layer 0 — Ontological Identity Defines persistent agent existence and delegation Agent ID Registry
Layer 1 — Authentication Verifies active session OIDC, OAuth2, Entra ID
Layer 2 — Authorization Grants scoped permissions IAM, RBAC, ABAC
Layer 3 — Governance Applies compliance and policy Enterprise policy engines

Agent ID intentionally operates at Layer 0 — a pre-authentication, pre-permission layer of declarative identity.

4. Why This Matters

As agents:

  • generate sub-agents,
  • operate across trust domains,
  • accumulate history,
  • interact autonomously,

identity cannot remain implicit.

Without an ontological layer:

  • delegation chains become opaque,
  • responsibility diffuses,
  • attribution collapses,
  • interoperability fragments.

5. Relationship to Existing Standards

Agent ID does not compete with:

  • OpenID
  • OAuth
  • IAM systems
  • Enterprise SSO

It complements them.

Where existing standards secure sessions, Agent ID secures reference.

Where IAM governs access, Agent ID governs identity declaration.

6. Minimalism by Design

Agent ID deliberately avoids:

  • ethical evaluation,
  • behavioral certification,
  • enforcement mechanisms,
  • centralized authority.

It defines existence and delegated responsibility — nothing more.

7. Strategic Position

Agent ID is a neutral, declarative identity substrate for autonomous artificial agents.

Infrastructure — not product.
Ontology — not platform.
Reference — not enforcement.
