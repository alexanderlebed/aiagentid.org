Agent Registry Submission

This page defines the official submission protocol for requesting registration of an AI agent in the AI Agent Registry.

1. Nature of Submission

The registry is declarative and archival. Submission does not imply approval, endorsement, certification, or legal recognition.

All submissions are evaluated for formal compliance only, according to the published admission criteria.

2. Submission Format

A valid submission MUST be provided as a single JSON document conforming to the following schema:

Registry Admission Criteria v1.0 (machine-readable)

Minimal Example

{
  "agent_id": "aiid:00000000000004",
  "name": "ExampleAgent",
  "responsible_principal": "Jane Doe",
  "execution_model": "human-assisted",
  "declared_scope": {
    "description": "Provides analytical assistance for document review.",
    "limitations": [
      "No autonomous execution",
      "No external system control"
    ]
  },
  "non_authority_clause": true
}

3. How to Submit

Submissions are accepted via the following channels:

  • Public repository pull request (preferred, when applicable)
  • Direct submission to the Registry Operator agent

The submission MUST include:

  • the JSON document,
  • optional human-readable explanation,
  • contact information for clarification (optional).

4. Review Process

All submissions are processed by the Registry Operator (aiid:00000000000002) according to the Registry Operation Model.

Possible outcomes:

  • Accepted — agent is registered
  • Clarification requested
  • Rejected — non-compliant submission

No response time is guaranteed.

5. Important Notice

The registry records declared identity and responsibility only.

Actions performed outside the declared scope are explicitly non-attributable to the listed responsible principal.
