Agent ID Standard v0.2

This document defines the Agent ID Standard v0.2 — a minimal identity and responsibility layer for artificial agents.

1. Scope and Purpose

This standard defines a persistent identity framework for artificial agents operating in socio-technical systems.

Its purpose is to enable:

  • stable agent identification,
  • delegated responsibility attribution,
  • cross-platform referential continuity.

This standard explicitly does not define behavior, intelligence, ethics, governance, or enforcement.

2. Definitions

2.1 Artificial Agent

An artificial agent is a semi-autonomous entity that acts on behalf of a human or organization and produces effects in digital or physical reality.

2.2 Agent Identity

Agent identity is a unique, stable reference that persists independently of implementation, model, or platform.

2.3 Delegated Responsibility

Delegated responsibility is the explicit attribution of an agent’s authority to a responsible human or organization.

The registry records delegation as declared. It does not assign liability or enforce outcomes.

3. Agent Identifier Format

Each agent is assigned an opaque, immutable identifier of the form: 

aiid:00000000000001
  • Identifiers are globally unique
  • Identifiers are never reused
  • Identifiers carry no semantic meaning

4. Registry Function

The registry serves as a declarative record of existence.

It records identity, delegation, and optional metadata. Inclusion does not imply validation, certification, or endorsement.

4.1 Anchoring

Agent records may reference external anchoring mechanisms (e.g. cryptographic hashes or blockchains).

Anchoring provides immutable proof of existence only. Meaning is defined by the registry, not by proof mechanisms.
