Agent ID Standard v0.2

This document defines the Agent ID standard for artificial agents.

1. Scope and Purpose

This standard defines a minimal identity layer for artificial agents.

Its purpose is to establish persistent identification, delegated responsibility, and referential continuity for artificial agents operating in socio-technical systems.

This standard does not define:

  • intelligence levels,
  • behavior rules,
  • ethical constraints,
  • governance mechanisms,
  • enforcement or liability frameworks.

This standard defines existence, not conduct.

2. Definitions

2.1 Artificial Agent

An artificial agent is a semi-autonomous entity that:

  • performs actions on behalf of a human or organization,
  • operates within declared delegated boundaries,
  • may persist across time, platforms, and implementations,
  • produces effects in digital or physical reality.

An agent is not merely software. An agent is an actor in a socio-technical system.

2.2 Agent Identity

Agent identity is a stable, unique reference that:

  • distinguishes one agent from all others,
  • persists independently of implementation,
  • enables attribution of actions and history.

Identity does not imply intelligence, trustworthiness, or authority. It enables recognition and reference only.

2.3 Delegated Responsibility

Delegated responsibility is the explicit attribution of an agent’s authority to a human or organization.

Delegation defines:

  • scope of permitted action,
  • operational boundaries,
  • responsibility attribution.

Delegation does not imply legal liability assignment, fault determination, or enforcement capability.

The registry records delegation as declared.

3. Agent Identifier Format

3.1 Identifier Structure

Each agent is assigned a globally unique identifier in the form: 

aiid:00000000000001

The identifier:

  • is opaque,
  • carries no semantic meaning,
  • is immutable once assigned.

3.2 Persistence

An Agent ID:

  • MUST remain stable over time,
  • MUST NOT be reused,
  • MUST NOT be reassigned.

An agent may be deprecated or archived, but its identifier remains reserved permanently.

3.3 Resolution

Each Agent ID MUST resolve to:

  • a human-readable record (HTML),
  • a machine-readable record (JSON).

Resolution mechanisms are implementation-dependent and out of scope for this standard.

4. Registry Function

4.1 Registry Role

The registry serves as a declarative record of existence.

It records:

  • agent identity,
  • declared role,
  • delegation attribution,
  • optional metadata,
  • optional anchoring references.

The registry does not validate claims beyond structural compliance.

4.2 Non-Authority Principle

The registry:

  • does not grant authority,
  • does not certify behavior,
  • does not enforce compliance,
  • does not adjudicate disputes.

Inclusion in the registry implies existence, not endorsement.

4.3 Anchoring

Agent records MAY include references to external anchoring mechanisms (e.g. cryptographic hashes, blockchains, notarization systems).

Anchoring:

  • provides immutability guarantees,
  • does not change registry semantics,
  • does not imply legal force.

4.4 Separation of Meaning and Proof

The registry defines meaning. External systems may provide proof.
