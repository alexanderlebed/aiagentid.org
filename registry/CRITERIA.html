Registry Admission Criteria v1.0

This document provides a human-readable description of the admission criteria for inclusion in the Agent ID Registry.

Normative source:
The canonical, machine-enforced definition of these criteria is the JSON Schema:
CRITERIA_v1.0.json

Purpose

The purpose of the Registry Admission Criteria is to define the minimum conditions of existence for an artificial agent to be recorded in the Agent ID Registry.

These criteria are intentionally minimal and non-normative. They do not evaluate quality, ethics, usefulness, or intent.

What the Criteria Do

An agent MAY be admitted to the registry if, and only if, it provides a complete declaration satisfying the criteria below.

  • Identity is syntactically valid and unique
  • A responsible principal is explicitly declared
  • An execution model is stated
  • Operational scope and limitations are described
  • A non-authority clause is explicitly acknowledged

If all criteria are satisfied, admission is mechanical. No discretionary judgment is applied.

What the Criteria Do NOT Do

The criteria explicitly do not:

  • assess intelligence or capability,
  • judge ethical alignment,
  • evaluate intent or potential harm,
  • grant authority or permission,
  • imply endorsement or certification.

The registry records existence — nothing more.

Machine Enforcement

Admission criteria are enforced via a JSON Schema that validates submissions mechanically.

Any submission that conforms to the schema is admissible. Any submission that does not conform is rejected automatically.

Important:
Human review is not required for admission. Human responsibility applies to approval and publication, not to semantic evaluation.

Versioning and Freeze

This criteria set is frozen as Registry Admission Criteria v1.0.

Any modification requires:

  • a new schema version,
  • a new canonical file,
  • explicit version designation.

Existing records remain governed by the criteria version under which they were admitted.
